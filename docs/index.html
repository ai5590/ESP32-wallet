Установка (только вручную)

Пока это демонстрационный MVP, поэтому установка идёт только из файлов `.zip`, которые лежат в этом же репозитории. Для работы нужно скачать архив, распаковать его в отдельную папку и подключить как локальное расширение.

Обновление (только вручную)

  1. Скачайте новый zip-файл с этой страницы.
  2. Распакуйте архив в новую папку (или обновите текущую).
  3. Откройте страницу расширений в браузере.
  4. Нажмите `Reload` для расширения или загрузите новую распакованную папку.

Пошаговая ручная установка

Chrome / Edge / Opera

  1. Скачайте `esp-wallet-chromium.zip` и распакуйте его.
  2. Откройте страницу расширений браузера.
  3. Включите режим разработчика.
  4. Нажмите Load unpacked.
  5. Выберите папку распакованной сборки Chromium.

Firefox

  1. Скачайте `esp-wallet-firefox.zip` и распакуйте его.
  2. Откройте about:debugging#/runtime/this-firefox.
  3. Нажмите Load Temporary Add-on....
  4. Выберите manifest.json из папки сборки Firefox.

О проекте

ESP32 Wallet — минималистичный Solana-кошелёк как браузерное расширение, совместимый с Wallet Standard и legacy window.solana. В текущей версии подпись выполняется локально в расширении.

Следующий этап: вынести хранение ключей и подпись на внешнее ESP32-устройство через отдельный transport-слой.

Что планируется дальше

Полноценный режим с ESP32

Подтверждение подписи будет переноситься на внешнее устройство ESP32. Тогда ключи остаются на устройстве, а в браузере работает только интерфейс и transport.

Единое устройство для разных программ

Идея — одно устройство подтверждения для разных сценариев: сайты через Wallet Standard/legacy connector, голосования DAO, отдельное десктоп/серверное приложение и другие интеграции.

Когда проект перейдёт в production-стадию, расширение можно опубликовать в сторах и пользователи получат установку в один клик. Для пользователей такой сценарий остаётся бесплатным.
